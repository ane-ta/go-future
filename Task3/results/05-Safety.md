# ADR-\_: Защита

## Контекст

Рост популярности сервиса увеличивает риск целенаправленных атак (DDoS, взлом БД через SQL-инъекции) и случайных утечек данных при межрегиональной репликации.

## Проблема

Снизить риск полной компрометации данных при обходе периметра.

## Решение

Эшелонированная защита (Defense in Depth)

### Поток запроса с защитой

1.  Пользователь → DDoS & WAF
    - Очистка трафика на границе сети

    1.  → DDoS Protection (фильтрация объёмных атак)
    2.  → WAF (фильтрация HTTP-запросов, SQL injection, XSS)
2.  → Regional firewall
    - Сетевая изоляция. Разрешает входящий трафик только от IP-адресов WAF.
3.  → API Gateway (rate limiting, аутентификация)
    - Прикладная защита (Rate Limiting, Auth)
4.  → Network Policies Kubernetes (Сетевые политики)
    - Микросегментация. Запрещает сервисам общаться с БД или друг другом без явного разрешения.
5.  → Микросервисы

### Межрегиональный трафик

Данные реплицируются через Transit Firewall, который инспектирует трафик между облаками и разрешает только протокол Kafka.

- MirrorMaker 2 (Kuala Lumpur) → Regional Firewall -\> Transit Firewall → Regional Firewal → Kafka (Singapore) (разрешено)
- Health checks → WAF -\> Regional Firewall → API Gateway (Singapore) (разрешено)
- Всё остальное запрещено

## Аргументация

### ➕ Скрытие Origin (инфраструктуры)

- **Можно:**
  - Предотвратить атаки напрямую на API Gateway
- **Так как:**
  - реальные IP-адреса серверов скрыты благодаря связке WAF + Firewall

### ➕ Минимизация «радиуса поражения»

- **Можно:**
  - Предотвратить ситуацию, когда через взлом микросервиса (например, «Уведомления») злоумышленник получает доступ к базе данных «Поездок».
- **Так как:**
  - Network Policies внутри K8s описывают внутренние разрешения взаимодействия

### ➕ Локализация трафика

- **Можно:**
  - чистить трафик максимально близко к пользователю, сохраняя низкую задержку
- **Так как:**
  - Используем Edge-узлы WAF

## Ограничения и риски

- Усложнение диагностики:
  - При возникновении ошибок 5xx теперь нужно проверять всю цепочку (WAF, FW, GW).
- Стоимость:
  - Глобальные сервисы защиты (Cloudflare/AWS Shield) увеличивают операционные расходы.
- Зависимость от провайдера:
  - Конфигурация WAF специфична для конкретного облачного вендора.

## Альтернативы

1.  VPN-only access:
    - Слишком медленно и неудобно для конечных пользователей такси.
2.  Самописные фильтры на уровне кода:
    - Неэффективно против объемных DDoS-атак и сложно в поддержке.
