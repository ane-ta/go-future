** Решение

Использовать следующие механизмы надежности взаимодействий
| Механизм                        | Реализация                                                                                | Где применяется                                |
| ------------------------------- | ----------------------------------------------------------------------------------------- | ---------------------------------------------- |
| **Idempotency**                 | `event_id` (UUID) в каждом событии; consumer хранит `processed_events`  (TTL = retention) | Все consumer'ы                                 |
| **Dead Letter Queue (DLQ)**     | Топик `dead.letter.queue` (1 партиция, retention 7 дней)                                  | Все consumer'ы                                 |
| **At-least-once + idempotency** | Брокер гарантирует доставку, consumer идемпотентен                                        | Все топики (exactly-once не требуется)         |
| **Transactional Outbox**        | Таблица `outbox_events` в каждой БД сервиса. CDC (Debezium)                               | Оркестратор (BookingCreated), Payments, Driver |
| **Circuit Breaker**             | Отключает вызов после N ошибок за период, fallback через DLQ | Внешние API (Payments → Яндекс.Пэй, Geo → Яндекс.Карты) |
| **Bulkhead**                    | Ограничение параллельных запросов (thread pool)              | Payments, Driver (защита от перегрузки)                 |
| **Timeout**                     | 5 секунд на вызов, при превышении → retry или отмена саги    | Все синхронные вызовы оркестратора                      |

**Transcactional outbox**
- Каждый сервис имеет таблицу =outbox_events= в своей БД
- Бизнес-данные и событие сохраняются в одной транзакции
- CDC (Debezium) читает WAL PostgreSQL и публикует события в брокер

**Idempotency**
Каждое событие содержит =event_id= (UUID). Consumer хранит =processed_events= в Redis (TTL 7 дней). При повторной доставке проверяет =event_id= и игнорирует дубли.

**Retry, Cirquit breaker и DLQ**
- Экспоненциальная задержка: 1 → 2 → 4 → 8 секунд, затем Cirquit breaker и DLQ
- Dead Letter Queue: топик =dead.letter.queue=, мониторинг размера (>100 → алерт)
- At-least-once + идемпотентность (exactly-once не требуется)

** Аргументация
**Transcactional outbox**
- Атомарность: событие опубликовано тогда и только тогда, когда данные сохранены
- Нет потери событий при падении сервиса
- Нет дополнительной нагрузки на БД (CDC читает WAL)
