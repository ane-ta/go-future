* Оркестратор
Синхронно
1. Проверить фрод
1. Заблокировать деньги
1. Создать заказ (статус PENDING_DRIVER)

Асинхронно
1. отслеживать таймаут (30 секунд на поиск водителя)
    1. Если никто не принял
        - отменить заказ и вернуть деньги
    1. Если водитель принял
        - зафиксировать назначение

Он отвечает за:

| Что делает оркестратор     | Как                                          |
| -------------------------- | -------------------------------------------- |
| Хранит состояние саги      | В своей БД (saga_id, шаг, статус)            |
| Вызывает сервисы синхронно | Fraud → Payments → Driver поиск (асинхронно) |
| Обрабатывает таймауты      | Если водитель не нашёлся за 30 сек → отмена  |
| Вызывает компенсации       | Если водитель не нашёлся → возврат денег     |
| Отвечает клиенту           | Через WebSocket / polling                    |

Клиент → API Gateway → Оркестратор
                           │
                           ├─ 1. Fraud.check() (синхронно)
                           │
                           ├─ 2. Payments.hold() (синхронно)
                           │
                           ├─ 3. Публикует OrderCreated
                           │      ↓
                           │   Driver Service ищет водителя (асинхронно)
                           │      ↓
                           │   Driver публикует DriverFound
                           │      ↓
                           │   Оркестратор получает, назначает
                           │
                           ├─ 4. Если таймаут → отмена, возврат денег
                           │
                           └─ 5. Клиент получает статус
