* Общие сведения о системе GoFuture
** Цель системы
Создание высокотехнологичной экосистемы для управления пассажирскими перевозками и логистикой. 
Система должна обеспечивать бесшовное взаимодействие между пассажирами, водителями и корпоративными клиентами в режиме реального времени.

** Ключевые бизнес-цели
1. *Масштабируемость*: Поддержка роста количества заказов (до 100к в сутки) без деградации производительности.
2. *B2B Экспансия*: Создание лучшего на рынке инструмента для управления корпоративными поездками.
3. *Автоматизация финансов*: Снижение нагрузки на бухгалтерию за счет автоматических выплат через Bank API.
4. *Гео-эффективность*: Минимизация времени подачи авто (ETA) до 5 минут в крупных городах.

** Стейкхолдеры и их интересы
| Стейкхолдер          | Роль в проекте                 | Основной интерес                         |
|----------------------+--------------------------------+------------------------------------------|
| Пассажиры            | Конечные пользователи B2C     | Скорость подачи, цена, удобство оплаты  |
| Водители             | Исполнители заказов           | Прозрачность баланса, быстрые выплаты    |
| Корп. менеджеры      | Клиенты B2B                   | Управление бюджетами, отчеты по поездкам |
| Бухгалтеры           | Операционный контроль         | Корректность реестров, сверка с банком   |
| Архитекторы/Dev      | Команда реализации            | Модульность кода, стабильность Monolith  |
| SRE-инженеры         | Команда поддержки             | Наблюдаемость (Metrics/Logs), Up-time    |

** Бизнес-ограничения
- *Локализация*: Соответствие регуляторным требованиям.
- *Стоимость*: Оптимизация использования облачных ресурсов (PostgreSQL RDS).
- *Внешние API*: Лимиты и стабильность Yandex Maps и платежных шлюзов.
